New to GitHub? Follow these steps — no tech skills needed

Create your free GitHub account & host your first site

Takes about 5 minutes. Every step explained in plain English.

Step 1

Create a free GitHub account

GitHub is like Google Drive for websites — and it's completely free

  1. Open a new browser tab and go to github.com
  2. Click the green "Sign up" button in the top-right corner
  3. Enter your email address, create a password, and choose a username
    (e.g. "johnsmith2024" — this will appear in your website address)
  4. Complete the short puzzle to verify you're human, then click "Create account"
  5. Check your email for a verification code from GitHub and enter it
  6. On the welcome questions, click "Skip personalisation" at the bottom
💡 Write down your GitHub username — you'll see it in your website address later: yourusername.github.io
Step 2

Create your first project folder (repository)

Think of a repository as an online folder where your website files live

  1. Once logged in to github.com, click the "+" icon (top-right) → "New repository"
  2. In the "Repository name" box, type a name using only lowercase letters and dashes
    Example: my-portfolio or my-business-site
  3. Make sure "Public" is selected (not Private)
  4. Tick the box that says "Add a README file"
  5. Scroll down and click the green "Create repository" button
💡 Your website address will be: yourusername.github.io/your-repository-name — so choose something you like!
Step 3

Turn your folder into a live website (GitHub Pages)

One setting flips your repository into a real, public website — for free

  1. Inside your new repository, click "Settings" (the gear icon near the top)
  2. Scroll down the left sidebar and click "Pages"
  3. Under "Branch", click the dropdown that says "None" and select "main"
  4. Leave the folder set to "/ (root)" and click "Save"
  5. Wait about 60 seconds, then refresh the page — you'll see a green box with your website link!
💡 It can take up to 2 minutes for your site to appear the first time. After that, updates are near-instant.
Step 4

Connect GitHub to Ready4Launch & build your app
